### Method 1: MONTESSORI — Self-Correcting Materials

**Core principle:** The material teaches the child — not the teacher. Wrong answers cause a natural consequence the child can see, not a voice saying "WRONG."

**Visual Example — Shape Sorter:**

```
┌─────────────────────────────────────────────────────────────────┐
│  MONTESSORI SHAPE SORTER                                        │
│                                                                 │
│  LOOSE PIECES:  △  ○  □  ⬟  ★  ●                               │
│                                                                 │
│  ┌──────────────────────────────────────────┐                  │
│  │  △ slot │  ○ slot │  □ slot │  ⬟ slot   │  ← Board         │
│  └──────────────────────────────────────────┘                  │
│                                                                 │
│  Child tries ○ in △ slot → piece BOUNCES BACK (self-corrects) │
│  Child tries ○ in ○ slot → CLICKS IN + chime sound            │
│                                                                 │
│  ✅ No voice says "WRONG" — the physical result teaches       │
│  ✅ Child feels success from OWN discovery, not from praise   │
│  ✅ Impossible to "fail" — just keep trying                   │
└─────────────────────────────────────────────────────────────────┘
```

**Games using this method:** Shape Sorter, Animal Habitat Sort, Color Bin Sort, Puzzle Assembly, Letter Tray, Number Rods

### Method 3: PHONICS — Sound-First Reading Foundation

**Core principle:** Letters are taught by their SOUNDS (/k/, /æ/, /t/) not their names ("see, ay, tee"). Children decode words by blending sounds.

**Visual Example — Phonics Flow:**

```
┌─────────────────────────────────────────────────────────────────┐
│  PHONICS LESSON — THE 3-STEP FLOW                              │
│                                                                 │
│  STEP 1 — INTRODUCE THE SOUND:                                 │
│  ┌───────────────────────────────────────┐                     │
│  │  Big letter "A" shown with sparkles   │                     │
│  │        🍎   A                         │                     │
│  │  Voice: "/æ/ ... /æ/ ... Apple!"      │                     │
│  └───────────────────────────────────────┘                     │
│                                                                 │
│  STEP 2 — DISCRIMINATION:                                      │
│  "Which one starts with /b/?"                                  │
│  Shows: 🍌  🐱  🎈  🐟                                        │
│  ← Child taps banana → ✅ "/b/... Banana! Great!"             │
│  ← Child taps cat → banana bounces to hint                    │
│                                                                 │
│  STEP 3 — WORD BUILDING (ages 4+):                            │
│  "Drag the sounds to spell CAT"                                │
│   /k/   /æ/   /t/   → [ C ] [ A ] [ T ] = CAT 🐱             │
│                                                                 │
│  ✅ Sound → Image → Word Building progression                 │
│  ✅ Never letter names — always sounds                        │
│  ✅ RV AppStudios only does step 1 & 2 — we do all 3         │
└─────────────────────────────────────────────────────────────────┘
```

**Games using this method:** Letter Sound Match, Word Builder, Rhyme Time, First Words, Story Time, Alphabet Puzzle

│                                                                 │
│  ┌────────────────────────────────────────────┐                │
│  │          ╔══╗                              │                │

│  └────────────────────────────────────────────┘                │
│                                                                 │

### Method 5: CONSTRUCTIVISM (Piaget) — Build Knowledge Step by Step

**Core principle:** New knowledge is built on top of existing knowledge. Each level unlocks only after the previous concept is secure.

**Visual Example — Number Concept Ladder:**

```
┌─────────────────────────────────────────────────────────────────┐
│  CONSTRUCTIVISM: THE NUMBER LADDER (Concept of "3")            │
│                                                                 │
│  RUNG 1 — CONCRETE:     🍎🍎🍎  → child TOUCHES 3 apples     │
│               ↓                                                 │
│  RUNG 2 — PICTURE:     [🍎][🍎][🍎] → tap to count           │
│               ↓                                                 │
│  RUNG 3 — SYMBOL:      ● ● ● = "3"  → dot to numeral match   │
│               ↓                                                 │
│  RUNG 4 — ABSTRACT:    1+2=?  2+1=?  3-0=?                    │
│               ↓                                                 │
│  RUNG 5 — APPLICATION: "You have 5 stars. You used 2.         │
│                          How many left?" (3)                   │
│                                                                 │
│  🚨 KEY RULE: Child CANNOT skip to Rung 3 without             │
│  completing Rungs 1 and 2. The app enforces the ladder.       │
│                                                                 │
│  RV AppStudios has no ladder — any activity anytime.          │
│  Our adaptive engine tracks which rung each child is on.      │
└─────────────────────────────────────────────────────────────────┘
```

### Method 7: SPACED REPETITION — Long-Term Memory Formation

**Core principle:** Reviewing content at increasing intervals moves it from short-term → long-term memory. Struggling words appear MORE; mastered words appear LESS.

**Visual Example — Word Review Schedule:**

```
┌─────────────────────────────────────────────────────────────────┐
│  SPACED REPETITION: "ELEPHANT" — Memory Journey                │
│                                                                 │
│  DAY 1:  Child sees Elephant first time in Animal World        │
│          → Brain: weak memory trace                            │
│  DAY 2:  Elephant appears in a sorting game                    │
│          → Brain: trace gets stronger                          │
│  DAY 4:  Elephant in a True/False statement                    │
│          → Brain: "I know this one!"                           │
│  DAY 8:  Elephant in a memory card flip                        │
│          → Brain: long-term storage forming                    │
│  DAY 16: Elephant in a puzzle                                  │
│          → Brain: MASTERED ✅                                  │
│                                                                 │
│  HARD WORDS (child got wrong):                                 │
│  Reappear every 1–2 days until mastered                       │
│                                                                 │
│  MASTERED WORDS:                                               │
│  Reappear only once per week to maintain memory               │
│                                                                 │
│  The system is INVISIBLE to the child.                        │
│  They never see "review mode" — it just feels like new games. │

### Method 8: SCAFFOLDED LEARNING (Vygotsky's ZPD) — The "Just Right" Challenge

**Core principle:** The Zone of Proximal Development (ZPD) = just above current ability + support. Too easy = boredom. Too hard = frustration. Scaffolding fills the gap.

**Visual Example — Puzzle Scaffold Ladder:**

```
┌─────────────────────────────────────────────────────────────────┐
│  SCAFFOLDING: PUZZLE HELP SYSTEM                               │
│                                                                 │
│  ATTEMPT 1: No help (try independently)                        │
│  ┌────────────────────┐                                        │
│  │  ? ? ?  ?  ?  ?   │  Child tries to place pieces           │
│  └────────────────────┘                                        │
│                                                                 │
│  ATTEMPT 2 WRONG: Subtle glow on correct piece                 │
│  ┌────────────────────┐                                        │
│  │  ✨?  ?  ?  ?  ?  │  ← one piece glows slightly           │
│  └────────────────────┘                                        │
│                                                                 │
│  ATTEMPT 3 WRONG: Arrow points to correct slot                 │
│  ┌────────────────────┐                                        │
│  │  ? ? → [SLOT]  ?  │  ← arrow nudges toward correct slot   │
│  └────────────────────┘                                        │
│                                                                 │
│  ATTEMPT 4 WRONG: Slot glows gold                              │
│  ATTEMPT 5 WRONG: Lumi auto-places + explains                 │
│                                                                 │
│  SCAFFOLD DECREASES as child advances:                         │
│  Easy level:  5 hints available                               │
│  Medium level: 3 hints available                              │
│  Hard level:  1 hint available                                │
│  Master level: 0 hints — pure independence                    │
└─────────────────────────────────────────────────────────────────┘

### Method 11: INQUIRY-BASED LEARNING — "What Do YOU Think?"

**Core principle:** Instead of giving answers, pose questions and let children hypothesize. Their predictions (right or wrong) create stronger memory than passive receiving.

**Visual Example — Color Mixing Inquiry:**

```
┌─────────────────────────────────────────────────────────────────┐
│  INQUIRY-BASED: COLOR MIXING LAB                               │
│                                                                 │
│  Step 1 — QUESTION:                                            │
│  Lumi: "What do you THINK will happen if we mix RED + BLUE?"   │
│                                                                 │
│  Step 2 — HYPOTHESIS:                                          │
│  Child sees 4 options: 🟣 🟢 🟠 🟤                            │
│  Child CHOOSES before seeing the answer                        │
│                                                                 │
│  Step 3 — EXPERIMENT:                                          │
│  🔴 + 🔵 pour into jar → slow mix animation → 🟣 PURPLE!     │
│                                                                 │
│  Step 4 — CONFIRM/CORRECT:                                     │
│  ✅ If right: "Amazing prediction! You're a scientist!"        │
│  ❌ If wrong: "Surprise! Red + Blue = Purple! Try another?"   │
│                                                                 │
│  Why this works:                                               │
│  The PREDICTION creates cognitive commitment.                 │
│  Whether right or wrong, the result is MEMORABLE.            │
│  "I thought it would be green but it was purple!" = strong    │
│  memory trace because of the surprise element.                │
└─────────────────────────────────────────────────────────────────┘
```

**Games using Inquiry:** Color Mixing Lab, What Am I? (guessing), Animal Diet guessing, If-Then concepts, True/False
